Overview

Project Overview

This project connects my IT support background with website maintenance thinking. It is not a live audit. It is a structured checklist showing what I would review when helping with a safer, cleaner WordPress redesign.

Problems

Common WordPress Problems

  • Too many inactive or overlapping plugins.
  • Outdated themes, plugins, or WordPress core files.
  • No clear backup and restore process.
  • Large images, heavy page builders, and unused scripts.

Security Checklist

Basic hardening items to review.

Updates

Confirm WordPress core, active theme, and plugins are maintained and updated safely.

Access

Review admin accounts, strong passwords, two-factor options, and least-privilege roles.

Backups

Check backup frequency, storage location, and whether restore steps are documented.

Forms

Confirm contact forms use spam protection and do not expose unnecessary data.

Performance

Performance Checklist

  • Compress large images and use suitable dimensions.
  • Review render-blocking CSS and JavaScript where possible.
  • Check lazy loading for below-the-fold images.
  • Review page builder sections that add weight without value.
  • Use PageSpeed Insights and Chrome DevTools for basic checks.
Plugin Hygiene

Plugin / Theme Checklist

  • Remove inactive plugins after confirming they are not needed.
  • Avoid duplicate plugins that solve the same problem.
  • Check last update date and compatibility before relying on a plugin.
  • Use a maintained theme and avoid editing theme core files directly.
  • Document important plugin settings before redesign work.

Recommendations

Before / After recommendation table.

Before After
Many plugins installed without clear purpose. Keep only necessary plugins and document what each one does.
No clear backup routine before edits. Create a backup checklist and test restore awareness before redesign.
Large images uploaded directly from design tools. Resize and compress images before upload, then test page speed.
Admin access is shared or unclear. Use named accounts, strong passwords, and appropriate roles.

Learning

What I Learned

WordPress performance and security are connected to process, not only visual redesign. A safer redesign needs update awareness, plugin review, backup planning, image discipline, and clear notes before making changes.
